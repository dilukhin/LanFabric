# LanFabric

LanFabric — это минималистичная система для создания защищённой виртуальной локальной сети (LAN) поверх интернета с использованием WireGuard.
Основная цель — дать возможность нескольким пользователям созавать окружение как в локальной сети, без предоставления доступа в интернет по умолчанию.

Проект ориентирован на:
- простоту развёртывания
- управляемость через CLI
- минимальные зависимости
- устойчивость к блокировкам (с возможностью расширения)

---

## Возможности

- создание VPN-сети уровня L3 (виртуальная локальная сеть)
- централизованное управление пользователями
- автоматическое назначение IP-адресов
- изоляция от интернета по умолчанию
- точечное включение интернета для конкретных пользователей
- работа через SSH без API и веб-интерфейсов
- хранение конфигурации в JSON
- логирование на сервере и клиенте
- поддержка нескольких серверов
- управление одной командой

---

## Архитектура

LanFabric состоит из двух компонентов:

1. vcli-admin.py — клиент управления
2. vsrv-admin.py — сервер управления

Схема работы:

  клиент (vcli-admin.py)
          |
         SSH
          |
  сервер (vsrv-admin.py)
          |
     WireGuard VPN
          |
      клиенты (игроки)

---

## Требования

### Сервер
- Ubuntu 22.04
- root-доступ по SSH
- доступ в интернет

### Клиент
- Windows 10 / Linux
- Python 3.12+
- установлен ssh и scp

---

## Установка и первый запуск

### 1. Подготовка

Скопируйте файлы:
- vcli-admin.py
- vsrv-admin.py

---

### 2. Инициализация сервера

Запустите:

  python vcli-admin.py init

Введите:
- имя сервера (локальное)
- IP или hostname
- SSH пользователя

Будет выполнено:
- установка WireGuard
- настройка VPN
- копирование серверного скрипта
- запуск сервера

---

## Работа с пользователями

### Создание пользователя

  python vcli-admin.py add-user <server> <username>

Результат:
- создаётся VPN-учётка
- генерируется конфигурация клиента

---

### Список пользователей

  python vcli-admin.py list <server>

---

### Удаление пользователя

  python vcli-admin.py delete-user <server> <username>

Требуется подтверждение.

---

### Блокировка пользователя

  python vcli-admin.py block-user <server> <username>

---

## Подключение клиента

Используется приложение WireGuard.

Шаги:
- установить WireGuard
- импортировать конфиг
- включить туннель

После подключения:
- все участники видят друг друга по IP вида 10.10.0.X

---

## Игра (Terraria)

Один игрок:
- запускает Host & Play

Остальные:
- подключаются по IP:
  10.10.0.X

---

## Интернет-доступ

По умолчанию интернет отключён.

Включается отдельно (будет реализовано отдельной командой).

---

## Хранение данных

### На сервере

  /opt/vpn-admin/
    users/
      <username>/
        user.json
        client.conf
    vpn-admin.log

---

### На клиенте

  ~/.vpn-admin/
    <server_name>/
      server.json

---

## Логирование

### Сервер
- файл: /opt/vpn-admin/vpn-admin.log
- используется logrotate:
  - ежедневная ротация
  - хранение 6 месяцев
  - сжатие старых логов

---

## IP-адресация

Сеть:
  10.10.0.0/24

Зарезервированы:
  10.10.0.1 — сервер
  10.10.0.2–10 — системные

Пользователи получают IP автоматически.

---

## Безопасность

- доступ к серверу только по SSH
- VPN-аутентификация только по ключам
- нет парольной аутентификации внутри VPN
- изоляция пользователей от интернета

---

## Relay (важно)

В некоторых сетях (редко) может потребоваться relay (L2 эмуляция).

Признаки:
- игроки не видят друг друга
- не работает автообнаружение
- подключение по IP не проходит

Проверка:
- попробуйте ping между клиентами
- попробуйте подключение по IP напрямую

В текущей версии relay НЕ реализован.

---

## Ограничения

- нет веб-интерфейса
- нет поддержки мобильных клиентов из коробки
- relay отсутствует
